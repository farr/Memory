\title{Constraining Gravitational Wave Memory with Hierarchical Inference}

\begin{abstract}
  With the multitude of gravitational wave observations that have been made in the past ten years, probing the dynamical and nonlinear nature of strong gravity is becoming more and more feasible. One promising way to test the nonlinear nature of Einstein's theory of general relativity (GR) is through the gravitational wave null memory effect: a nonlinear prediction of GR which corresponds to initially co-moving observers being permanently displaced due to a burst of gravitational radiation. Previous studies have shown that, while it is unlikely that the memory effect will be observed in a single event by the LIGO-Virgo-KAGRA (LVK) detectors, evidence for memory in the population of LVK events should be attainable after $\sim$2,000 gravitational wave detections. These works, however, relied on Bayes factors to perform their memory population analyses: an approach that can depend sensitively on the analysis priors and, when naively multiplied across many events, can even favor incorrect conclusions. In this work, using the GWTC-4.0 catalog of binary black hole observations, we instead perform hierarchical Bayesian inference---which is not subject to the issues associated with Bayes factors---to measure the evidence for memory in current LVK observations. We find that we can constrain what we call the memory enhancement factor---the constant appearing in front of the contribution to the strain from the supermomentum flux---to \muLambdaconstraint, consistent with its expected value of 1. In broad agreement with prior works, we forecast that $\sim$\hatAnumberofevents detections will be needed to constrain the uncertainty in the memory enhancement factor to $\lesssim1$.
\end{abstract}

\section{Introduction}
\label{sec:introduction}

A key motivation for gravitational wave (GW) science is to probe Einstein's theory of General Relativity (GR) and test whether or not it accurately describes all of the gravitational interactions that we observe in nature. While GR has been remarkably successful so far~\cite{Will:2014kxa,LIGOScientific:2026qni,LIGOScientific:2026fcf,LIGOScientific:2026wpt}, it is largely believed to be an incomplete theory, in part because it is not currently known how to reconcile it with quantum field theory in an ultraviolet-complete way~\cite{Goroff:1985th}. According to effective field theory, beyond-GR effects are expected to scale with the spacetime curvature~\cite{Donoghue:2017ovt,Burgess:2020tbq}. Consequently, a particularly promising avenue for testing GR is through its nonlinear predictions, since these are most sensitive to regimes of strong spacetime curvature.

Among the various nonlinear predictions of GR, the GW null memory effect is an especially alluring one~\cite{Christodoulou:1991cr,Blanchet:1992br,PhysRevD.45.520}. The GW memory effect corresponds to the permanent net displacement that two initially co-moving observers will experience due to a burst of gravitational radiation.\footnote{While there are also many other types of GW memory effects or, more generally, ``persistent observables'', in this work since we will exclusively focus on the displacement memory, we will refer to this effect broadly as ``memory'' or the ``memory effect''~\cite{Cachazo:2014fwa,Pasterski:2015tva,Nichols:2018qac,Flanagan:2018yzh,Flanagan:2019ezo,Grant:2021hga,Grant:2023ged}.} Its ordinary form was originally found in Ref.~\cite{Zeldovich:1974gvh} in 1974 by studying hyperbolic black hole scattering, while its null form is largely attributed to having been discovered in Ref.~\cite{Christodoulou:1991cr}, but was also noticed earlier in Refs.~\cite{Payne:1983rrr,Blanchet1990HDR}.\footnote{Originally the two contributions to the memory were called ``linear'' and ``nonlinear'' (or ``Christodoulou''). However, they have since been renamed to ``ordinary'' and ``null'' to better reflect the way in which they are sourced~\cite{Bieri:2013ada}; see, e.g., Ref.~\cite{Mitman:2024uss} for a review.} The ordinary contribution to the memory is sourced by unbound sources and exists in linearized GR, while the null contribution is sourced by gravitational radiation interacting with itself and is thus inherently nonlinear.

 Apart from its impact on observers, the memory effect is also intriguing because it is intimately connected to asymptotic symmetries~\cite{Newman:1966ub,Ashtekar:1981bq,Geroch:1981ut,Dray:1984rfa,Ludvigsen:1989kg,Flanagan:2014kfa} and soft theorems~\cite{Weinberg:1965nx,Strominger:2013jfa,Strominger:2014pwa}. As a result, memory plays an important role in fixing the coordinate freedom of gravitational systems~\cite{Mitman:2021xkq,Mitman:2022kwt}, formulating the concept of celestial holography~\cite{Strominger:2017zoo,Raclariu:2021zjz,Pasterski:2021raf,Pasterski:2021rjz}, and even in working toward establishing a divergence-free and self-consistent theory of quantum gravity~\cite{Prabhu:2022zcr}.

Unfortunately, because the memory effect is largely a low-frequency effect, it is nearly impossible to detect in individual binary black hole (BBH) observations with the current LIGO-Virgo-KAGRA (LVK) GW detectors~\cite{Lasky:2016knh}.  Nonetheless, Ref.~\cite{Lasky:2016knh} showed that one can measure the memory in a population of observations by performing a coherent summation of sub-threshold signals, which is often referred to as ``signal-to-noise ratio (SNR) stacking''. Assuming a population of GW150914-like events, which at the time was the only direct GW observation that had been made, Ref.~\cite{Lasky:2016knh} showed that 35 (90) events are needed to make a combined memory SNR of 3 (5). Later, once a more informative GW population had been established by the LVK detectors, Refs.~\cite{Hubner:2019sly,Boersma:2020gxx,Hubner:2021amk,Grant:2022bla,Cheung:2024zow}\footnote{Note that in an earlier version of the \texttt{gwmemory} package used by some of these works, a bug was present that led to the incorrect calculation of the memory~\cite{Talbot:2018sgr}. This has since been fixed.} conducted various follow-up studies on memory using certain LVK catalogs up to the GWTC-3.0 catalog and concluded that memory had yet to be confidently measured in the population of GW events and would require $\mathcal{O}(2$,$000)$ events to do so. All of these analyses, however, relied on Bayes factors to perform their inference---an approach that tends to be sensitive to prior choices and can yield misleading results when naively combined across many events~\cite{Isi:2022cii}---and did not simultaneously infer the memory alongside the population of other astrophysical parameters, which is crucial for performing a consistent analysis.

In this work, we do exactly that. Namely, we utilize hierarchical Bayesian inference to simultaneously infer the evidence for the memory in the LVK Collaboration's GWTC-4.0 catalog of GW observations as well as the usual astrophysical parameter populations. The paper is outlined as follows. In Sec.~\ref{sec:memory} we review how the memory can be computed from the gravitational wave strain and how this will be relevant to our analysis. In Sec.~\ref{sec:methodology} we illustrate our methodology for including memory in the analysis of the GWTC-4.0 catalog and performing hierarchical inference to constrain memory across the population of GW events. In Sec.~\ref{sec:results} we provide our various results, with Sec.~\ref{sec:memoryconstraints} showing the constraints on the memory population obtained via hierarchical inference and Sec.~\ref{sec:forecasts} showing a forecast for how many GW events are required to measure memory. And, finally, in Sec.~\ref{sec:discussion} we summarize our findings and highlight prospects for observing the memory effect in the near future.

\section{Computation of memory}
\label{sec:memory}

Until recently, numerical relativity (NR) simulations of BBH mergers could not resolve the memory effect~\cite{Favata:2010zu,Pollney:2010hs,Mitman:2020pbt}. This was largely because the way the strain waveforms were extracted to future null infinity in NR---a technique known as extrapolation~\cite{Boyle:2019kee}---was insufficient for resolving the low-frequency memory effect.\footnote{The more robust method of using Cauchy-characteristic evolution, or CCE for short, for obtaining waveforms at future null infinity, however, does resolve memory effects~\cite{Bishop:1996gt,Bishop:1997ik,Bishop:1998uk,Reisswig:2009rx,Reisswig:2009us,Babiuc:2010ze,Pollney:2010hs,Handmer:2014qha,Handmer:2015dsa,Handmer:2016mls,Moxon:2020gha,Moxon:2021gbv,Mitman:2020pbt}.} As a result of this--- at least in part---the majority of waveform models do not contain memory~\cite{Taracchini:2013rva,Pan:2013rra,Hannam:2013oca,Bohe:2016gbl,Ossokine:2020kjp,Varma:2019csw,Pratten:2020ceb,Colleoni:2024knd,Pompili:2023tna,Ramos-Buades:2023ehm,Hamilton:2021pkf,Thompson:2023ase}.\footnote{See Ref.~\cite{Yoo:2023spi} for a model trained on CCE waveforms which does contain memory effects.} Fortunately, due to the way in which memory is sourced, for any model of the gravitational wave strain, the memory can be computed from that model via a kind of post-processing~\cite{Talbot:2018sgr,Mitman:2020pbt,Mitman:2020bjf}. 
\begin{widetext}
\noindent In particular, motivated by the BMS symmetry group and its corresponding conserved charges~\cite{Wald:1999wa,Flanagan:2015pxa,Mitman:2020pbt}, one can write the real part of the gravitational wave strain---which is the only part that contains memory---as
\begin{align}
	\label{eq:restrain}
	\mathrm{Re}\left[h(u,\theta,\phi)\right]=\bar{\eth}^{2}\mathfrak{D}^{-1}\left[m+\frac{1}{4}\int_{-\infty}^{u}|\dot{h}|^{2}\,du\right],
\end{align}
where $\eth$ is the usual spin-weight operator~\cite{Newman:1961qr,Goldberg:1966uu,Geroch:1973am}, whose action on spin-weighted spherical harmonics~\cite{Goldberg:1966uu} is
\begin{align}
	\eth\phantom{}_{s}Y_{\ell,m}=\frac{1}{\sqrt{2}}\sqrt{(\ell-s)(\ell+s+1)}\phantom{}_{s+1}Y_{\ell,m},
\end{align}
$\mathfrak{D}$ is an operator, whose action on spherical harmonics is
\begin{align}
	\mathfrak{D}Y_{\ell,m}=\frac{1}{8}(\ell+2)(\ell+1)\ell(\ell-1)Y_{\ell,m},
\end{align}
$m$ is the Bondi mass aspect~\cite{Bondi:1962px}, and the coordinate $u\equiv t-r$ is the Bondi time (or the retarded time) with overdots representing derivatives with respect to $u$. Thus, since the null memory contribution to the strain is sourced by the supermomentum flux, i.e., the second term in Eq.~\eqref{eq:restrain}, one can compute the memory as
\begin{align}
	\label{eq:memory}
	h_{\mathrm{mem}}(u,\theta,\phi)=\frac{1}{4}\bar{\eth}^{2}\mathfrak{D}^{-1}\int_{-\infty}^{u}|\dot{h}|^{2}\,du.
\end{align}
Critically, the computation of $h_{\mathrm{mem}}$ requires knowledge of $h$ over the entire two-sphere. To express $h_{\mathrm{mem}}$ in terms of the angular basis of spin-weighted spherical harmonics, one can use the integral formula
\begin{align}
	\int_{S^{2}}\phantom{}_{s_{1}}Y_{\ell_{1},m_{1}}\phantom{}_{s_{2}}Y_{\ell_{2},m_{2}}\phantom{}_{s_{3}}Y_{\ell_{3},m_{3}}\,d\Omega=\sqrt{\dfrac{(2\ell_{1}+1)(2\ell_{2}+1)(2\ell_{3}+1)}{4\pi}}\begin{pmatrix}\ell_{1}&\ell_{2}&\ell_{3}\\m_{1}&m_{2}&m_{3}\end{pmatrix}\begin{pmatrix}\ell_{1}&\ell_{2}&\ell_{3}\\-s_{1}&-s_{2}&-s_{3}\end{pmatrix}
\end{align}
where the right-most objects are the Wigner-3j symbols with $s_{1}+s_{2}+s_{3}=0$, to obtain the memory contribution to the individual $(\ell,m)$ spin-weight -2 modes as
\begin{align}
	\label{eq:memorymodes}
	h_{\mathrm{mem}}^{\ell,m}(u)&=\sum\limits_{\substack{\ell_{1},\ell_{2}\\m_{2}}}(-1)^{m_{2}}\sqrt{\dfrac{(2\ell_{1}+1)(2\ell_{2}+1)(2\ell+1)}{4\pi(\ell+2)(\ell+1)\ell(\ell-1)}}\begin{pmatrix}\ell_{1}&\ell_{2}&\ell\\m-m_{2}&m_{2}&-m\end{pmatrix}\begin{pmatrix}\ell_{1}&\ell_{2}&\ell\\-2&2&0\end{pmatrix}\int_{-\infty}^{u}\dot{\overline{h}}_{\ell_{1},m-m_{2}}\dot{h}_{\ell_{2},m_{2}}\,du.
\end{align}
\end{widetext}
Therefore, given a waveform model which provides the spin-weight -2 modes of the strain, one can compute the memory contribution to each mode using Eq.~\eqref{eq:memorymodes}.

An important caveat regarding calling $h_{\mathrm{mem}}$ in Eq.~\eqref{eq:memory} the ``memory'', however, is that formally the memory is really only defined as a difference in the strain between the two boundaries of future null infinity, i.e., between $u\rightarrow-\infty$ and $u\rightarrow+\infty$. As a result, how one should represent the time evolution of the memory is not obvious. For example, one could naturally take the time evolution of the memory 
\begin{enumerate}
	\item to be a step-function, in which case it would have a strict $1/f$ dependence in the frequency domain;
	\item to be that of Eq.~\eqref{eq:memory}, which although agrees with the memory when taking its difference between the boundaries $u\rightarrow-\infty$ and $u\rightarrow+\infty$ may include non-memory content in the intermediate regime;
	\item or, to be some kind of Isaacson average of Eq.~\eqref{eq:memory}, which assumes the memory's time evolution should be monotonic, which is also not obvious~\cite{Zosso:2026czc}.
\end{enumerate}

For simplicity, and to avoid taking averages, in this study we take choice (ii) to be the proxy for the memory's time evolution. Consequently, we can aim to constrain the memory by considering the strain model
\begin{align}
	\label{eq:strainmodel}
	h(u;&\,\lambda,A_{\mathrm{mem}})=\nonumber\\
	&h_{\mathrm{no\,mem}}(u;\lambda) + A_{\mathrm{mem}}\,h_{\mathrm{mem}}(u;\lambda),
\end{align}
for $\lambda$ a vector of the 17 intrinsic and extrinsic parameters\footnote{Because we lack a waveform model that includes the eccentricity and mean anomaly, $\lambda$ is really a 15-dimensional vector.} and $A_{\mathrm{mem}}$ the ``memory enhancement factor'', which is 1 in GR. Equation~\eqref{eq:strainmodel} can then be used to measure $A_{\mathrm{mem}}$ across the population of GW observations, by taking $h_{\mathrm{no\,mem}}$ to be the waveform used in the LVK analysis, i.e., that of Refs.~\cite{Varma:2019csw,Pratten:2020ceb,Colleoni:2024knd,Pompili:2023tna,Ramos-Buades:2023ehm,Hamilton:2021pkf,Thompson:2023ase}, and $h_{\mathrm{mem}}$ to be Eq.~\eqref{eq:memory} (or really the sum of modes from Eq.~\eqref{eq:memorymodes}) using $h=h_{\mathrm{no\,mem}}$.
	
\section{Methodology}
\label{sec:methodology}

\figMemoryPosteriors

As stated in Sec.~\ref{sec:memory}, the LVK Collaboration's analysis of their catalog of events does not use waveform models that contain memory. For this analysis, the log-likelihood used to produce the public posteriors of parameters $\lambda$ takes the form
\begin{align}
	\log\mathcal{L}&=-\frac{1}{2}\langle d- R(\lambda)h(\lambda)|d-R(\lambda)h(\lambda)\rangle+C\nonumber\\
	&=-\frac{1}{2}\langle d-\tilde{h}(\lambda)|d-\tilde{h}(\lambda)\rangle+C\nonumber\\
	&=-\frac{1}{2}||d-\tilde{h}(\lambda)||^{2}+C
\end{align}
where $d$ is the observed data, $h(\lambda)$ is the waveform model evaluated at parameters $\lambda$, $R(\lambda)$ is the detector response (including calibration effects), $\tilde{h}(\lambda)\equiv R(\lambda)h(\lambda)$ is $h(\lambda)$ projected into the data using the detector response $R(\lambda)$, $\langle\cdot|\cdot\rangle$ is the noise-weighted inner product ( $||\cdot||^{2}\equiv\langle\cdot|\cdot\rangle$), and $C$ is some unimportant parameter-independent constant. Let $r(\lambda)\equiv d-\tilde{h}(\lambda)$ denote the residual between the data and the waveform projected into the data using the detector response. With $r(\lambda)$, the likelihood is simply
\begin{align}
	\log\mathcal{L}=-\frac{1}{2}||r(\lambda)||^{2}+C.
\end{align}
If we wish to include the effects of memory, as was outlined in Sec.~\ref{sec:memory}, since the memory can be computed directly from the usual waveform models using Eqs.~\eqref{eq:memorymodes} and~\eqref{eq:memory}, the likelihood becomes
\begin{align}
	\log\mathcal{L}=-\frac{1}{2}|| r(\lambda)-\tilde{h}_{\mathrm{mem}}(\lambda)||^{2}+C
\end{align}
and requires no new waveform model. But, if we wish to instead \emph{detect} the memory, then we want to include the memory enhancement factor via
\begin{align}
	\label{eq:memoryLL}
	\log\mathcal{L}=-\frac{1}{2}|| r(\lambda)-A_{\mathrm{mem}}\tilde{h}_{\mathrm{mem}}(\lambda)||^{2}+C.
\end{align}
Computing the posterior on $A_{\mathrm{mem}}$ allows us to easily test the consistency of the observed memory effect with the predictions of GR, and to determine whether the data prefer a non-zero memory effect at all. Conveniently, Eq.~\eqref{eq:memoryLL} takes the form of a Gaussian on $A_{\mathrm{mem}}$, which peaks at the value
\begin{align}
	\label{eq:mean}
	\hat{A}_{\mathrm{mem}}=\dfrac{\mathrm{Re}\left[\langle\tilde{h}_{\mathrm{mem}}(\lambda)|r(\lambda)\rangle\right]}{||\tilde{h}_{\mathrm{mem}}(\lambda)||^{2}},
\end{align}
with standard deviation
\begin{align}
	\label{eq:stddev}
	\sigma_{A_{\mathrm{mem}}}=||\tilde{h}_{\mathrm{mem}}(\lambda)||^{-1}.
\end{align}
Marginalizing out the memory enhancement factor of the likelihood with a completely agnostic (improper) flat prior yields the marginal likelihood
\begin{align}
	\label{eq:marginalLL}
	\log\overline{\mathcal{L}}&=\log\int\exp(\log\mathcal{L})\,dA_{\mathrm{mem}}\nonumber\\&=-\frac{1}{2}\left(||r(\lambda)||^{2}-\sigma_{A_{\mathrm{mem}}}^{-2}\hat{A}_{\mathrm{mem}}^{2}\right)\nonumber\\
	&\phantom{=.}+\log\sigma_{A_{\mathrm{mem}}}+C.
\end{align}
The sample-level hyperparameters $\hat{A}_{\mathrm{mem}}$ and $\sigma_{A_{\mathrm{mem}}}$ yield the preferred $A_{\mathrm{mem}}$ value (through $\hat{A}_{\mathrm{mem}}$) as well as our uncertainty about this value (through $\sigma_{A_{\mathrm{mem}}}$). Meanwhile, the marginal likelihood $\overline{\mathcal{L}}$ from tells us how much better the data is explained when we include a memory signal in the analysis with the usual waveform model evaluation. Therefore, if we have a set of samples drawn from a posterior based on the original, no-memory likelihood $\mathcal{L}$, then the ratio of the marginal likelihood in Eq.~\eqref{eq:marginalLL} to the original likelihood serves as an importance weight that can be applied to the samples to obtain a new set of samples drawn from the posterior that is based on the marginal likelihood $\overline{\mathcal{L}}$, i.e.,
\begin{align}
	\label{eq:weight}
	\log w&=\log\overline{\mathcal{L}}(\lambda)-\log\mathcal{L}(\lambda)\nonumber\\&=\frac{1}{2}\sigma_{A_{\mathrm{mem}}}^{-2}\hat{A}_{\mathrm{mem}}^{2}+\log\sigma_{A_{\mathrm{mem}}}.
\end{align}
If the memory is not very strong in any individual event, then the importance weights will be very close to 1, and the re-weighted samples will remain very similar to the original samples. Furthermore, each sample can also be augmented by a value of $A_{\mathrm{mem}}$ that is drawn from the Gaussian distribution described by Eqs.~\eqref{eq:mean} and~\eqref{eq:stddev} to obtain a new set of samples that were drawn from the posterior on $A_{\mathrm{mem}}$. With the values of $\hat{A}_{\mathrm{mem}}$ and $\sigma_{A_{\mathrm{mem}}}$ obtained for each sample, we can then combine them in a hierarchical way to obtain an overall posterior on the population distribution of $A_{\mathrm{mem}}$ across the entire catalog of GW events to determine whether the data prefer a non-zero memory effect in the population as a whole.

\subsection{Hierarchical Inference}
\label{sec:hierarchicalinference}

Following the work of Ref.~\cite{Payne:2023kwj}, we simultaneously model the memory and all of the usual BBH merger astrophysical parameters. We assume that we start from regular parameter estimation samples $\lambda_{s}$ that have been enhanced to include both $\hat{A}_{\mathrm{mem}}$ and $\sigma_{A_{\mathrm{mem}}}$, which we will hereafter call $\mu_{s}$ and $\sigma_{s}$ when referring to the sample $s$. This naturally defines the conditional distribution on the memory enhancement factor $A_{\mathrm{mem}}$---which we now call $A$ to shorten notation---given $\lambda_{s}$, which is
\begin{align}
	\label{eq:Aconditionaldistribution}
	p(A|\lambda_{s})=\mathcal{N}(A|\mu_{s},\sigma_{s}).
\end{align}
The hierarchical likelihood for population parameters $\Lambda$ for the $i^{\mathrm{th}}$ event requires computing the integral
\begin{align}
	\label{eq:hierarchicalLL}
	\mathcal{L}_{i}=\int\dfrac{p(A,\lambda)}{W(\lambda)}p(A,\lambda|\Lambda)\,dAd\lambda,
\end{align}
where $p(A,\lambda)$ is the sample posterior and $W(\lambda)$ is the importance weight that encodes the sampling prior and any necessary Jacobians as well as the importance weight that was derived in Eq.~\eqref{eq:weight}; we will assume that the prior on $A$ is flat so the weight $W$ is independent of $A$. Because $p(A,\lambda)=p(A|\lambda)p(\lambda)$, we can write Eq.~\eqref{eq:hierarchicalLL} as
\begin{align}
	\mathcal{L}_{i}=\int \dfrac{p(A|\lambda)p(\lambda)}{W(\lambda)}p(A,\lambda|\Lambda)\,dAd\lambda,
\end{align}

At this point, we can now use the fact that since we have $N$ samples from the posterior $\lambda_{s}$, i.e.,
\begin{align}
	p(\lambda)\approx\frac{1}{N}\sum\limits_{s}\delta(\lambda-\lambda_{s})
\end{align}
we can approximate the integral as
\begin{align}
	\mathcal{L}_{i}&\approx\frac{1}{N}\sum\limits_{s}\int\dfrac{p(A|\lambda)}{W(\lambda)}p(A,\lambda|\Lambda)\delta(\lambda-\lambda_{s})\,dAd\lambda\nonumber\\
	&=\frac{1}{N}\sum\limits_{s}\int\dfrac{p(A|\lambda_{s})}{W(\lambda_{s})}p(A,\lambda_{s}|\Lambda)\,dA.
\end{align}
If we further assume that the population model factorizes as $p(A,\lambda_{s}|\Lambda)=p(A|\Lambda_{A})p(\lambda_{s}|\Lambda_{\lambda})$ where $\Lambda_{A}$ and $\Lambda_{\lambda}$ are the memory and astrophysical hyperparameters, then
\begin{align}
	\label{eq:hierarchicalLLsimplified}
	\mathcal{L}_{i}\approx\frac{1}{N}\sum\limits_{s}\dfrac{p(\lambda_{s}|\Lambda_{\lambda})}{W(\lambda_{s})}\int p(A|\lambda_{s})p(A|\Lambda_{A})\,dA.
\end{align}
And, finally, if we assume a Gaussian population model on A with mean $\mu_{\Lambda}$ and standard deviation $\sigma_{\Lambda}$ so that $p(A|\Lambda_{A})=\mathcal{N}(A|\mu_{\Lambda},\sigma_{\Lambda})$, then the integral over $A$ in Eq.~\eqref{eq:hierarchicalLLsimplified} can be performed analytically, yielding
\begin{align}
	\label{eq:hierarchicalLLfinal}
	\mathcal{L}_{i}&\approx\frac{1}{N}\sum\limits_{s}\dfrac{p(\lambda_{s}|\Lambda_{\lambda})}{W(\lambda_{s})}\int\mathcal{N}(A|\mu_{s},\sigma_{s})\mathcal{N}(A|\mu_{\Lambda},\sigma_{\Lambda})\,dA\nonumber\\
	&=\frac{1}{N}\sum\limits_{s}\dfrac{p(\lambda_{s}|\Lambda_{\lambda})}{W(\lambda_{s})}\mathcal{N}\left(\mu_{s}|\mu_{\Lambda},\sqrt{\sigma_{s}^{2}+\sigma_{\Lambda}^{2}}\right),
\end{align}
where we used Eq.~\eqref{eq:Aconditionaldistribution} to replace $p(A|\lambda_{s})$ in Eq.~\eqref{eq:hierarchicalLLsimplified}. Equation~\eqref{eq:hierarchicalLLfinal} thus provides a straightforward way to evaluate the hierarchical likelihood \emph{without sampling the latent variable $A$} for each sample. Instead, each sample contributes to the likelihood through a Gaussian term that compares its sample-level memory mean $\mu_{s}$ (Eq.~\eqref{eq:mean}) with the memory population mean $\mu_{\Lambda}$, with a variance that is a sum in quadrature of the sample-level memory measurement uncertainty $\sigma_{s}$ (Eq.~\eqref{eq:stddev}) and the memory population uncertainty $\sigma_{\Lambda}$. Finally, while Eq.~\eqref{eq:hierarchicalLLfinal} is the likelihood for an individual event, note that we can obtain the likelihood across all events trivially as
\begin{align}
	\log\mathcal{L}=\sum\limits_{i}\log\mathcal{L}_{i}.
\end{align}
This enables a measurement of $\mu_{\Lambda}$ and $\sigma_{\Lambda}$ across the population of BBH observations via hierarchical inference. The population mean $\mu_{\Lambda}$ provides the preferred value of the memory enhancement factor $A$ across the population, while $\sigma_{\Lambda}$ tells us our uncertainty about the value of $A$. In the following section, we outline our procedure for performing this analysis and quote our findings for these two quantities in relation to their GR values of 1 and 0.

\section{Results}
\label{sec:results}

For our memory analysis, we consider all BBH events observed by the LVK Collaboration which had an inverse false alarm rate above one year, up to and including the GWTC-4.0 catalog. This amounted to \Nmemoryevents events in total. For each event, each available waveform model, and its corresponding posterior samples, we computed the memory mean $\mu_{s}$ and standard deviation $\sigma_{s}$ using Eqs.~\eqref{eq:mean} and~\eqref{eq:stddev}. The waveform models used are those described in Refs.~\cite{Varma:2019csw,Pratten:2020ceb,Colleoni:2024knd,Pompili:2023tna,Ramos-Buades:2023ehm,Hamilton:2021pkf,Thompson:2023ase}. Because the computation of the memory requires the strain modes (see Eq.~\eqref{eq:memorymodes}), each waveform model was always used to provide as many modes as available. For the phenomenological models, which output waveforms in the frequency domain, we first transformed the output waveform to the time domain before computing the memory. In Fig.~\ref{fig:MemoryPosteriors} we show the distribution of $\mu_{s}$ and $\sigma_{s}$ across all posterior samples for six of the more informative and popular BBH detections. As shown (especially by the third row of the figure), of these more significant events, none of them yield a confident measurement of memory and every event has a preferred memory enhancement factor that is consistent with the GR value of 1. Across all of the analyzed events, we find that GW230814~\cite{LIGOScientific:2025cmm} has the most influence on the inference of the memory in the sense that when we remove this event from the hierarchical inference the posterior on $(\mu_{\Lambda},\sigma_{\Lambda})$ changes the most. Note that while in Fig.~\ref{fig:MemoryPosteriors} we show results for the event GW250114~\cite{LIGOScientific:2025rid,LIGOScientific:2025wao}, this event is not included in our hierarchical inference since the sensitivity estimates have yet to be released by the LVK Collaboration via their GWTC-5.0. As we will show in the subsequent section, even though no single event yields a confident measurement of memory, by instead inferring the memory hyperparameters across all events, we can obtain much better memory constraints.

\subsection{Memory Constraints}
\label{sec:memoryconstraints}

With the computation of $\mu_{s}$ and $\sigma_{s}$ performed for the posterior samples of all events and all of the available waveform models, we performed two hierarchical analyses: one in which we measured the memory hyperparameters $\mu_{\Lambda}$ and $\sigma_{\Lambda}$ (see Eq.~\eqref{eq:hierarchicalLLfinal}) independent of all of the other astrophysical parameters, and one in which we measured everything simultaneously.\footnote{Note that, for the joint inference, the selection criterion restricting the secondary mass to be $>3M_{\odot}$ reduced our total number of events to \Nevents (instead of \Nmemoryevents).} When conducting these analyses, we opted to use the following hierarchy for which waveform model to use. If available, we  ideally used the NR surrogate model, then the EOB models, and then the Phenom models.\footnote{For certain samples, we found that the waveforms produced by the Phenom models had unphysical spikes at low frequencies that caused incredibly large $\mu_{s}$ values. Because of this, when using the Phenom models, if the value of $|\mu_{s}/\sigma_{s}|$ was $>10$ for any sample, we simply removed that sample from the analysis. This amounts to removing only a few tens of samples from the whole analysis.} This choice was made based on the model accuracy results presented in Fig.~2 of Ref.~\cite{LIGOScientific:2025rsn}. When we used an EOB model, we used \texttt{SEOBNRv5PHM} if available, and otherwise \texttt{SEOBNRv4PHM}~\cite{Bohe:2016gbl,Ossokine:2020kjp,Pompili:2023tna,Ramos-Buades:2023ehm}. When we used a Phenom model, we used \texttt{IMRPhenomXO4a} if available, and otherwise \texttt{IMRPhenomPHM\_SpinTaylor}, or \texttt{IMRPhenomPHM}~\cite{Pratten:2020ceb,Colleoni:2024knd,Hamilton:2021pkf,Thompson:2023ase}. We used the same population model as was used in Ref.~\cite{LIGOScientific:2025pvj}.

Our results for this study are highlighted in Fig.~\ref{fig:TGR}, which shows the $1\sigma$, $2\sigma$, and $3\sigma$ credible regions for the posterior of $\mu_{\Lambda}$ and $\sigma_{\Lambda}$. As can be seen, across all of the available events that we analyzed, we can currently constrain the memory $\mu_{\Lambda}$ and $\sigma_{\Lambda}$ hyperparameters to \muLambdaconstraint and \sigmaLambdaconstraint, respectively. Consequently, we find that, as was expected, with the current catalog of GW observations we cannot claim a detection of the memory as $\mu_{\Lambda}$ is not strongly bounded away from 0.

\subsection{Forecasts}
\label{sec:forecasts}

\figForecast

To forecast the number of events that will be needed to constrain $\mu_{\Lambda}$ to be away from zero, i.e., to confidently measure memory, we require a representation of each event that is compatible with the hierarchical likelihood that was derived in Eq.~\eqref{eq:hierarchicalLLfinal}. To do this, we compress each current event $i$ to an effective Gaussian approximation. Specifically, starting from the weighted representation
\begin{align}
	p(A|d_{i})\propto\sum\limits_{s}w_{s}\mathcal{N}(A|\mu_{s},\sigma_{s}),
\end{align}
we define an effective event center $\mu_{i}$ as the mode of this mixture and an effective local width $\sigma_{i}$ from the curvature at the mode via
\begin{align}
	\label{eq:effectivesigma}
	\sigma_{i}^{-2}=-\frac{\partial^{2}}{\partial A^2}\log p(A|d_{i})\Big|_{A=\mu_{i}}.
\end{align}
This is required because obtaining $\sigma_{i}$ this way preserves the local information content of the event, whereas using a typical sample value of $\sigma_{s}$ would not, since it ignores the overall spread in $\mu_{s}$. Now, as outlined in Appendix A of Ref.~\cite{Isi:2022cii}, the uncertainty on $\mu_{\Lambda}$---denoted as $\Delta\mu_{\Lambda}$---for some set of event-level uncertainties $\{\sigma_{i}\}_{i}^{N}$ is\footnote{Really, Appendix A of Ref.~\cite{Isi:2022cii} stated the homoskedastic version of this in their Eq.~(A8), but the heteroskedastic version shown in our Eq.~\eqref{eq:heteroskedasticmean} is straightforward to work out.}
\begin{align}
	\label{eq:heteroskedasticmean}
	\Delta\mu_{\Lambda}=\left[\sum\limits_{i}\left(\sigma_{i}^{2}+\sigma_{\Lambda}^2\right)^{-1}\right]^{-\frac{1}{2}}.
\end{align}
Consequently, to ensure that the $\sigma_{i}$ we produce using Eq.~\eqref{eq:effectivesigma} match our current $\Delta\mu_{\Lambda}$ according to Eq.~\eqref{eq:heteroskedasticmean}, we apply a global calibration factor $\alpha$ to all $\sigma_{i}$ as $\sigma_{i}\rightarrow\alpha\sigma_{i}$ so that our current $\Delta\mu_{\Lambda}$ is indeed reproduced via Eq.~\eqref{eq:heteroskedasticmean}. Note though that this calibration has a minimal impact, as the $\alpha$ we apply is \alphavalue, which is close to 1.

With our $\sigma_{i}$ for all current observations known, we can estimate the expected information per future event as
\begin{align}
	\label{eq:futureinfo}
	\overline{I}_{\mu,\mathrm{f}}(g,\sigma_{\Lambda})=\frac{1}{N}\sum\limits_{i=1}^{N}\left[\left(\sigma_{i}/g\right)^{2}+\sigma_{\Lambda}^{2}\right]^{-1},
\end{align}
i.e., take the average information of a future observation to be the average of every current observation's information. In Eq.~\eqref{eq:futureinfo}, $\sigma_{\Lambda}$ is taken to be some draw from our current $\sigma_{\Lambda}$ posterior conditioned on $\mu_{\Lambda}=1$ and $g$ is some gain to account for the event-level uncertainty decreasing with future detector upgrades. For example, we could take $g=1.41$ to account for the increased sensitivity between the O4a and O5a runs~\cite{IGWN:ObservingCapabilities:2026}. Consequently, we find that after $N_{\mathrm{f}}$ events, our information on $\Delta\mu_{\Lambda}$ is
\begin{align}
	I_{\mu,\mathrm{total}}(N_{\mathrm{f}},g,\sigma_{\Lambda})=I_{\mu,\mathrm{current}}+N_{\mathrm{f}}\overline{I}_{\mu,\mathrm{f}}(g,\sigma_{\Lambda}),
\end{align}
i.e.,
\begin{align}
	\label{eq:uncertaintyestimator}
	\Delta\mu_{\Lambda}(N_{\mathrm{f}},g,\sigma_{\Lambda})=\left[I_{\mu,\mathrm{total}}(N_{\mathrm{f}},g,\sigma_{\Lambda})\right]^{-\frac{1}{2}}.
\end{align}
Equation~\eqref{eq:uncertaintyestimator} provides a Fisher-matrix motivated way to estimate the number of future events required to reach some uncertainty $\Delta\mu_{\Lambda}$ provided some sensitivity gain $g$ and population-wide $\sigma_{\Lambda}$, which again can simply be drawn from the current $\sigma_{\Lambda}$ posterior.

The results of this forecast analysis are shown in Fig.~\ref{fig:Forecast}. Each curve corresponds to computing $\Delta\mu_{\Lambda}$ through Eq.~\eqref{eq:uncertaintyestimator} for some future number of GW observations. The blue curve represents O4a-like observations ($g=1$), while yellow represents O5a-like observations ($g=1.41$). Bands around each curve show the half-widths obtained across \NSigmaDraws $\sigma_{\Lambda}$ values, taken as draws from the current $\sigma_{\Lambda}$ posterior. The horizontal axes on top of the figure show the time that is required to reach the number of observations shown on the lower horizontal axis for the events rates of 180 and 500 per year for O4a and O5a~\cite{IGWN:ObservingCapabilities:2026}. As can be seen, this analysis yields a very similar number of GW observations required to constrain memory as a naive $1/\sqrt{N}$ estimate, i.e., $\sim$\hatAnumberofevents events. But, if the sensitivity of the LVK detectors in future runs meets the expected O5a gain of $\sim1.41$, then due to the increase in individual event SNRs, we may be able to obtain evidence for memory in as little as $\sim$1,000 events.\footnote{Note though that this is a bit misleading because as the detectors become more sensitive, they are expected to observe many more low-SNR events (as well as more high-SNR events). A better way to view the sensitivity gain is that we will reach $\sim$2,000 events \emph{much faster} with an O5a-like detector, than an O4a-like one.}

\section{Discussion}
\label{sec:discussion}

Gravitational wave memory is a key prediction of GR that is a natural target for ground-based GW detectors. It is of immense theoretical interest due to its connection to asymptotic symmetries and soft theorems and for its role in formulating a consistent theory of quantum gravity. Unfortunately, due to its low-frequency nature, it is an incredibly challenging effect to observe in individual events with the current LVK detectors. However, there is reason to believe that with enough observations, we can observe memory in a population of GW detections.

In this study, we used hierarchical Bayesian inference to measure the memory enhancement factor---the constant appearing in front of the contribution to the strain from the supermomentum flux, which is 1 in GR---across the population of GW events up to and including the recent GWTC-4.0 catalog. Unsurprisingly, we found that we can constrain this factor to \muLambdaconstraint, which does not yield a confident measurement of the memory effect. Nonetheless, this work provides the first attempt to measure memory using hierarchical inference and thus the first analysis to successfully account for potential changes in the inferred BBH population due to the memory. Like previous works, we find that we will likely need $\sim$\hatAnumberofevents GW detections to confidently constrain the memory away from zero. Yet, since this number is expected to be achieved with the fifth observing run of the LVK detectors, we find that it is likely that we will measure the memory effect for the first time in a population of events in the next few years.
